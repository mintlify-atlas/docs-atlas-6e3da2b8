Name paints are a cosmetic reward for 7TV subscribers. When a user who has an active paint sends a message, their username is rendered with the paint's style instead of their plain Twitch username color. Chatterino7 renders paints natively — no browser extension needed.

![Example of name paints rendering in Chatterino7](https://user-images.githubusercontent.com/27637025/227034147-cb1fcd76-dbae-4878-9551-96ffa64dd1a9.png)

## Paint types

7TV defines three paint functions, all of which Chatterino7 supports:

| Type | Description |
|------|-------------|
| `LINEAR_GRADIENT` | A gradient swept at a configurable angle across the username. Supports hard color stops and repeating patterns. |
| `RADIAL_GRADIENT` | A radial (circular) gradient emanating from the center of the username text. Supports repeating patterns. |
| `URL` | An image fetched from a URL and used as the fill texture for the username. |

All three types can include one or more **drop shadows** — configurable x/y offsets, blur radius, and RGBA color — applied behind the text.

## How paints are loaded

Paints are received via the **7TV EventAPI** WebSocket connection. When Chatterino7 subscribes to a channel, 7TV sends `COSMETIC_CREATE` dispatches for each paint used by users in that channel. These are parsed and stored in a paint registry keyed by paint ID.

When a user sends a message, Chatterino7 looks up their Twitch or Kick username in the paint map and retrieves the associated `Paint` object. The paint is then used to produce a `QPixmap` for the username text using Qt's gradient and brush APIs, composited at render time.

When 7TV assigns or revokes a paint via an `ENTITLEMENT_CREATE` or `ENTITLEMENT_DELETE` event, the paint map is updated immediately and all open channel view buffers are invalidated so the change appears without restarting.

## Drop shadows

Each paint can carry an array of drop shadow descriptors with the following properties:

- **x_offset** / **y_offset** — horizontal and vertical displacement in pixels
- **radius** — blur radius
- **color** — RGBA color of the shadow

Drop shadow radii in Chatterino7 match the browser extension's rendering as of version 7.5.5-beta.1.

### Disabling paint shadows

If you find paint shadows distracting or visually noisy, you can turn them off:

1. Go to **File → Settings**.
2. Open the **7TV** tab.
3. Uncheck **Enable paint drop shadows**.

<Note>
  This setting was added in Chatterino7 **7.5.2-beta.1**.
</Note>

## Behavior in system messages

Paints are **not rendered in system messages** — for example, sub notifications, raid announcements, or channel point redemption notices. This is intentional: system messages are generated by Twitch rather than by the user, and applying a paint there could cause visual confusion. Only regular chat messages authored by the user display their paint.

## Real-time paint updates

Because paints are delivered over the EventAPI, they update without any action on your part:

- A new paint entitlement granted to a user appears within seconds.
- If a user's paint is revoked, their username returns to its normal color immediately.
- Paint assignments are tracked separately for Twitch and Kick connections, so the correct paint is shown regardless of which platform the message arrives from.

## Settings reference

| Setting | Location | Description |
|---------|----------|-------------|
| Enable paint drop shadows | **Settings → 7TV** | Toggles rendering of drop shadows on painted usernames. |
